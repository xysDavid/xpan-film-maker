# 电影画幅照片生成器

## 项目概述

构建一个 Node.js 命令行工具，将哈苏 XPan 全景照片（65:24 比例，约 2.7:1）转换为电影画幅风格的图片，并添加中英双语字幕。

## 输入 / 输出

- **输入**：从 `./input` 目录读取所有图片文件（`.jpg`、`.jpeg`、`.png`、`.tiff`）。
- **输出**：将处理后的图片以 `.jpg` 格式（质量 90）写入 `./output` 目录。输出文件名保持原始文件名不变。

## 处理流程

### 第一步：补黑边至 16:9

XPan 比例（约 2.7:1）比 16:9（1.778:1）更宽。为了达到 16:9：

1. 保持原图宽度不变。
2. 计算目标画布高度：`目标高度 = 宽度 / (16 / 9)`。
3. 创建一个尺寸为 `宽度 x 目标高度` 的新画布，填充**纯黑色**（`#000000`）。
4. 将原始照片在画布上**水平居中**。**垂直方向**上，将照片放置在略高于中心的位置（距顶部约 45%），使底部黑色区域大于顶部——为下方字幕留出更多空间。

### 第二步：添加中英双语字幕

在**底部黑色区域**（原始照片下方）放置电影风格的双语文字（中文 + 英文）：

#### 文字布局

```
┌─────────────────────────────────────────────────┐
│                 （顶部黑色区域）                    │
├─────────────────────────────────────────────────┤
│                                                 │
│              原始 XPan 照片                       │
│                                                 │
├─────────────────────────────────────────────────┤
│                （底部黑色区域）                     │
│                                                 │
│              这是中文字幕示例                      │
│          This is an English subtitle             │
│                                                 │
└─────────────────────────────────────────────────┘
```

#### 字体排版规则

- **中文文字**：白色（`#FFFFFF`），较大字号，位于上行。
- **英文文字**：白色（`#FFFFFF`）或略暗的白色（`#CCCCCC`），较小字号（约为中文字号的 75%），位于中文下方。
- 两行文字均**水平居中**。
- 中英文之间的行距：约为中文字号的 0.4 倍。
- 字幕整体（两行合在一起）在底部黑色区域内**垂直居中**。
- 字号应根据图片宽度自适应缩放（例如：中文字号 ≈ 宽度 / 55，英文字号 ≈ 宽度 / 70）。可根据实际效果微调。

#### 字体要求

- 中文：使用简洁的无衬线中文字体。优先使用项目内置字体以保证可移植性（如 Noto Sans SC、思源黑体）。如无内置字体则回退到系统字体。
- 英文：使用简洁的无衬线英文字体（如同款 Noto Sans 或专用的西文字体）。
- 支持从 `./fonts` 目录加载自定义 `.ttf` / `.otf` 字体文件。

### 第三步：输出

- 格式：JPEG，质量 90。
- 输出目录：`./output`（如不存在则自动创建）。

## 配置

使用 `config.json` 文件（或命令行参数）定义每张图片或全局的字幕文案：

```jsonc
{
  // 全局默认字幕（应用于没有单独配置的所有图片）
  "defaultSubtitle": {
    "cn": "默认中文字幕",
    "en": "Default English Subtitle"
  },
  // 按输入文件名（不含扩展名）单独配置的字幕
  "images": {
    "DSC00123": {
      "cn": "东京的黄昏",
      "en": "Dusk in Tokyo"
    },
    "DSC00456": {
      "cn": "京都的雨季",
      "en": "Rainy Season in Kyoto"
    }
  }
}
```

## 技术栈

- **运行环境**：Node.js（>=18）
- **图片处理**：[sharp](https://sharp.pixelplumbing.com/) —— 高性能原生图片处理库。
- **文字渲染**：使用 `sharp` 的 composite 功能配合 SVG 文字叠加，或使用 [@napi-rs/canvas](https://github.com/nicedoc/napi-rs-canvas) / [node-canvas](https://github.com/Automattic/node-canvas) 进行精确的文字绘制。选择能可靠渲染中日韩文字的方案。
- **无 GUI**：纯命令行/脚本工具。

## 使用方法

```bash
# 安装依赖
npm install

# 将照片放入 ./input，编辑 config.json，然后运行：
node src/index.js

# 或通过命令行参数覆盖字幕（应用于所有图片）：
node src/index.js --cn "电影感的一天" --en "A Cinematic Day"
```

## 边界情况与其他要求

1. **无文字模式**：如果某张图片没有配置字幕（也没有默认字幕），则只生成补黑边后的画幅，不添加文字。
2. **宽高比容错**：如果输入图片不是标准 XPan 比例，仍然正常处理——将任意比例的图片补黑边至 16:9。如果输入图片比 16:9 更窄（更高），则在左右两侧补黑边，字幕仍放在底部。
3. **大文件处理**：能高效处理高分辨率输入（如宽度 7000+ 像素）。尽可能使用流式处理。
4. **批量处理**：对 `./input` 中的所有图片进行顺序处理，或使用受控并发（如同时处理 3 张）。
5. **覆盖保护**：如果输出文件已存在，默认覆盖；可通过 `--no-overwrite` 参数跳过已存在的文件。
6. **进度反馈**：在终端打印处理进度（如 `[2/10] 正在处理 DSC00123.jpg...`）。
