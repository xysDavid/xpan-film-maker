# 横向图片裁切 - 3:4 竖向分割

将一张横向图片裁切为多张宽高比 3:4 的竖向图片，适用于社交媒体多图发布。

## 你的任务

根据用户的指令裁切图片。用户输入如下：

$ARGUMENTS

## 执行步骤

### 1. 检查环境

确认插件目录下 `node_modules` 存在，如果不存在则在插件根目录运行 `npm install` 安装依赖。

### 2. 解析用户意图

用户的输入可能是以下几种形式，请灵活识别：

- **指定图片路径**：如 `/path/to/image.jpg` 或 `./input/photo.png`
  → 提取输入图片路径
- **指定输出目录**：如 `输出到 /path/to/output` 或 `保存到 ./result`
  → 提取输出目录路径
- **仅给文件名**：如 `裁切 photo.jpg`
  → 在 input 目录中查找该文件

### 3. 使用 MCP 工具或 CLI

**方式 A - 使用 MCP 工具 `crop_image_3x4`（推荐）：**

如果 xpan-film MCP 工具可用，直接调用：

```
crop_image_3x4({
  inputPath: "/输入图片的绝对路径",
  outputDir: "/输出目录的绝对路径"
})
```

**方式 B - 使用 CLI 命令：**

在插件根目录下运行：

```bash
node src/crop-3x4.js <输入图片路径> [输出目录]
```

示例：
```bash
node src/crop-3x4.js ./input/photo.jpg ./output/cropped
```

### 4. 报告结果

处理完成后，告诉用户：
- 原图尺寸
- 裁切为多少张图片
- 每张图片的尺寸
- 丢弃了多少像素
- 输出文件保存在哪个目录

## 注意事项

- 输入图片必须是横向的（宽 > 高），否则会报错
- 支持的输入格式：`.jpg` `.jpeg` `.png` `.tiff` `.tif`
- 输出格式：JPEG 质量 90
- 裁切逻辑：高度保持不变，每张宽度 = 高度 × 3/4，左右均匀丢弃多余像素
- 输出文件命名：`{原文件名}_01.jpg`, `_02.jpg`, ...
